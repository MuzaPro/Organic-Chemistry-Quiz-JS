Code Style Guidelines

Contents

Introduction

Consistent code style throughout the Organic Chemistry Quiz project helps to maintain readability, reduce errors, and make collaboration easier. This document outlines the coding standards and practices that all contributors should follow.

Following these guidelines ensures that the codebase remains consistent, maintainable, and easier for all contributors to understand.

HTML Guidelines

  • Use semantic HTML elements
  • Include appropriate ARIA attributes for accessibility
  • Keep markup clean and properly indented
  • Use HTML5 doctype
<section class="quiz-section">
  <h2>Question</h2>
  <div class="question-content" aria-live="polite">
    <p id="question-text">Question text goes here</p>
  </div>
</section>

Best Practices

  • Use classes for styling and JavaScript hooks
  • Add descriptive alt text to images
  • Group related form elements with fieldset
  • Keep HTML structure logically nested
  • Use comments for complex sections

CSS Guidelines

  • Use the BEM (Block, Element, Modifier) naming convention
  • Organize CSS by component
  • Use CSS variables for theming
  • Include responsive designs with media queries

BEM Naming Convention

/* BEM Naming Example */
.question-card {}           /* Block */
.question-card__title {}    /* Element */
.question-card__content {}  /* Element */
.question-card--highlighted {} /* Modifier */

Using CSS Variables

/* Variables in :root */
:root {
  --primary-color: #4361ee;
  --secondary-color: #f8f9fa;
}

/* Using variables */
.button {
  background-color: var(--primary-color);
}

Media Queries

/* Mobile-first approach */
.container {
  width: 100%;
  padding: 1rem;
}

/* Tablet breakpoint */
@media (min-width: 768px) {
  .container {
    width: 750px;
    padding: 1.5rem;
  }
}

/* Desktop breakpoint */
@media (min-width: 1200px) {
  .container {
    width: 1170px;
    padding: 2rem;
  }
}

Best Practices

  • Use relative units (em, rem) for scalability
  • Comment complex or non-intuitive CSS
  • Minimize specificity conflicts
  • Group related properties
  • Maintain a consistent order of properties

JavaScript Guidelines

  • Use ES6+ features
  • Follow the Module Pattern for encapsulation
  • Use descriptive variable and function names
  • Include JSDoc comments for functions
  • Handle errors appropriately

Module Pattern

const ModuleName = (() => {
  // Private variables
  let privateVar = 'value';
  
  // Private functions
  function privateFunction() {
    // Implementation
  }
  
  // Public API
  return {
    publicMethod: function() {
      // Use private variables/functions
      return privateFunction();
    },
    
    anotherPublicMethod: function(param) {
      // Implementation
    }
  };
})();

JSDoc Comments

/**
 * Check if the answer is correct
 * @param {Array} userAnswer - Array of reagent IDs
 * @returns {boolean} Whether the answer is correct
 */
const checkAnswer = (userAnswer) => {
  try {
    const currentQuestion = questions[currentQuestionIndex];
    const correctAnswer = currentQuestion.correctReagents;
    
    // Sort both arrays to ensure order doesn't matter
    const sortedUserAnswer = [...userAnswer].sort();
    const sortedCorrectAnswer = [...correctAnswer].sort();
    
    return JSON.stringify(sortedUserAnswer) === JSON.stringify(sortedCorrectAnswer);
  } catch (error) {
    console.error('Error checking answer:', error);
    return false;
  }
};

Error Handling

function playSound(soundName) {
  try {
    if (!sounds[soundName]) {
      throw new Error(`Sound "${soundName}" does not exist`);
    }
    
    const sound = sounds[soundName];
    sound.currentTime = 0;
    
    // Return a promise to handle mobile autoplay restrictions
    return sound.play()
      .catch(error => {
        console.warn(`Sound playback failed: ${error.message}`);
        // Graceful fallback - continue without sound
      });
  } catch (error) {
    console.error('Error in playSound:', error);
    // Return a resolved promise for consistent API
    return Promise.resolve();
  }
}

Variable Naming

  • Use camelCase for variables and functions
  • Use PascalCase for constructor functions and classes
  • Use UPPER_SNAKE_CASE for constants
  • Use descriptive names that indicate purpose

Best Practices

  • Avoid global variables
  • Use const and let instead of var
  • Use arrow functions for callbacks
  • Chain promises with .then() or use async/await
  • Destructure objects and arrays when appropriate
  • Use template literals for string concatenation
  • Use optional chaining and nullish coalescing when appropriate

Documentation Guidelines

Code Comments

  • Use JSDoc comments for functions
  • Add inline comments for complex logic
  • Keep comments current with code changes
  • Use comments to explain "why", not "what"

Feature Documentation

  • Create feature documentation for significant features
  • Include usage examples
  • Document limitations and known issues
  • Provide integration instructions

Good documentation makes your code more maintainable and helps other developers understand your thought process.

Accessibility Guidelines

  • Use semantic HTML elements
  • Include appropriate ARIA attributes
  • Ensure proper focus management
  • Provide text alternatives for non-text content
  • Ensure sufficient color contrast
  • Support keyboard navigation

Focus Management Example

// When opening a modal
function openModal(modalElement) {
  // Store the current active element to restore focus later
  previouslyFocused = document.activeElement;
  
  // Make modal visible
  modalElement.classList.add('visible');
  
  // Find the first focusable element and focus it
  const firstFocusable = modalElement.querySelector(
    'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
  );
  
  if (firstFocusable) {
    firstFocusable.focus();
  }
}

// When closing a modal
function closeModal(modalElement) {
  // Hide modal
  modalElement.classList.remove('visible');
  
  // Restore focus to the previously focused element
  if (previouslyFocused) {
    previouslyFocused.focus();
  }
}
