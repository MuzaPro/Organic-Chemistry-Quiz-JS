Development Environment Setup

Contents

Introduction

The Organic Chemistry Quiz JS project uses a lightweight development environment focused on modern web technologies. This guide will walk you through setting up your development environment for contributing to the project.

The project is designed to be easy to set up and work with, requiring minimal dependencies and configuration.

Required Software

Essential Tools

  1. Git
    • Download Git
    • Used for version control and collaboration
    • Recommended to use Git Bash on Windows
  2. Code Editor
    • Visual Studio Code (recommended)
    • Alternatives: Sublime Text, Atom, WebStorm
    • Make sure your editor supports HTML, CSS, and JavaScript
  3. Web Browser
    • Modern browser with good developer tools (Chrome, Firefox, Edge)
    • Chrome is recommended for its comprehensive DevTools

Recommended VS Code Extensions

  • Live Server - For local development with auto-reload
  • ESLint - JavaScript linting
  • Prettier - Code formatting
  • HTML CSS Support - Enhanced HTML/CSS editing
  • SVG Preview - For viewing SVG molecule files
  • Git Lens - Enhanced Git capabilities
# Installing VS Code extensions from command line
code --install-extension ritwickdey.LiveServer
code --install-extension dbaeumer.vscode-eslint
code --install-extension esbenp.prettier-vscode
code --install-extension ecmel.vscode-html-css
code --install-extension jock.svg
code --install-extension eamodio.gitlens

Setting Up Your Local Environment

1. Clone the Repository

# Clone the repository
git clone https://github.com/muzapro/Organic-Chemistry-Quiz-JS.git

# Navigate to the project directory
cd Organic-Chemistry-Quiz-JS

2. Configure VS Code (Optional but Recommended)

Create a .vscode folder with the following files for consistent settings:

settings.json

{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "html.format.wrapLineLength": 100,
  "css.validate": true,
  "javascript.format.semicolons": "insert",
  "javascript.preferences.quoteStyle": "single",
  "editor.tabSize": 2,
  "files.eol": "\n",
  "files.insertFinalNewline": true,
  "files.trimFinalNewlines": true
}

extensions.json

{
  "recommendations": [
    "ritwickdey.LiveServer",
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "ecmel.vscode-html-css",
    "jock.svg",
    "eamodio.gitlens"
  ]
}

3. Set Up Live Server

  1. Install the Live Server extension in VS Code
  2. Right-click on index.html in the project root
  3. Select "Open with Live Server"
  4. Your project should now open in your default browser

Live Server will automatically reload the page when you save changes to your files, making development faster and more efficient.

Optional Development Tools

Linting and Formatting

For a more robust development setup, you can add ESLint and Prettier configuration:

1. Install Node.js

If you want to use ESLint and Prettier for better code quality:

  • Download Node.js (LTS version recommended)

2. Install ESLint and Prettier

# Navigate to project directory
cd Organic-Chemistry-Quiz-JS

# Initialize npm (if not already done)
npm init -y

# Install ESLint and Prettier
npm install --save-dev eslint prettier eslint-config-prettier eslint-plugin-prettier

3. Create ESLint Configuration

Create a .eslintrc.json file in the project root:

{
  "env": {
    "browser": true,
    "es2021": true
  },
  "extends": ["eslint:recommended", "plugin:prettier/recommended"],
  "parserOptions": {
    "ecmaVersion": 12,
    "sourceType": "module"
  },
  "rules": {
    "no-unused-vars": "warn",
    "no-console": "warn",
    "prettier/prettier": "warn"
  }
}

4. Create Prettier Configuration

Create a .prettierrc file in the project root:

{
  "singleQuote": true,
  "trailingComma": "es5",
  "printWidth": 100,
  "tabWidth": 2,
  "semi": true
}

Browser Developer Tools

Browser developer tools are essential for web development. Here's how to make the most of them:

Chrome DevTools

  1. Open DevTools
    • Windows/Linux: Press F12 or Ctrl+Shift+I
    • Mac: Press Command+Option+I
    • Or right-click on the page and select "Inspect"
  2. Key Panels
    • Elements: Inspect and modify the DOM and CSS
    • Console: View logged messages and run JavaScript code
    • Network: Monitor network requests
    • Application: Inspect local storage and session storage
    • Performance: Analyze runtime performance
    • Sources: Debug JavaScript code

Mobile Testing

Use the device emulation feature in Chrome DevTools to test mobile compatibility:

  1. Open DevTools
  2. Click the "Toggle device toolbar" button (or press Ctrl+Shift+M)
  3. Select a device from the dropdown or set custom dimensions

When testing the drag-and-drop functionality on desktop in mobile emulation mode, use Chrome's touch emulation for more accurate testing.

Working with SVG Files

The project uses SVG files for molecular structures. Here are some tools to help work with them:

Recommended SVG Editors

  • Inkscape - Free, open-source vector graphics editor
  • Figma - Web-based design tool with SVG support
  • Vecta.io Nano - Simple online SVG editor

SVG Optimization

Before adding new SVG files to the project, optimize them to reduce file size:

  • SVGOMG - Online SVG optimizer
  • Command line: SVGO
# Install SVGO globally
npm install -g svgo

# Optimize an SVG file
svgo molecule.svg -o molecule.min.svg

Troubleshooting Common Issues

Live Server Not Starting

  • Check if the port is already in use (default: 5500)
  • Try changing the Live Server port in VS Code settings
  • Restart VS Code

CORS Errors When Loading JSON

  • Make sure you're running the project through a server (like Live Server)
  • Check browser console for specific error messages
  • Ensure paths to resources are correct and consistent

Git Issues

  • Make sure you have the correct permissions for the repository
  • Use git status to check for uncommitted changes
  • Use git pull to sync with the latest changes
