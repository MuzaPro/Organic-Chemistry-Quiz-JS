Testing Guidelines

Contents

Introduction to Testing

Thorough testing is essential to ensure the Organic Chemistry Quiz functions correctly across all supported devices and browsers. This document outlines our testing approach and provides guidelines for contributors.

All new features and bug fixes should be thoroughly tested before submitting a pull request. This helps maintain the quality and reliability of the application.

Testing Levels

The Organic Chemistry Quiz project employs several levels of testing:

1. Manual Testing

The primary method for validating functionality, UI interactions, and visual appearance.

  • Functional testing of quiz mechanics
  • UI/UX testing
  • Cross-browser compatibility
  • Responsive design verification
  • Accessibility testing

2. Console-Based Testing

Using browser developer tools to validate code execution and catch errors.

  • Checking for JavaScript errors
  • Verifying proper data loading
  • Testing state management

3. Automated Testing (Future Implementation)

Plans for implementing automated tests using Jest and Puppeteer are in progress.

  • Unit tests for core functions
  • Integration tests for component interactions
  • End-to-end tests for complete user journeys

Manual Testing Checklist

Follow this checklist when testing changes to ensure a consistent experience:

General Application Testing

  1. Application Startup
    • Application loads without errors
    • Welcome screen displays correctly
    • Start button functions properly
  2. Quiz Navigation
    • Question navigation buttons work correctly
    • Question counter displays accurately
    • Progress indicator updates appropriately
  3. Quiz Functionality
    • Questions display correctly with all required elements
    • Reagents appear in the reagent area
    • Drop zones are clearly marked
    • Score tracking functions properly
  4. Feedback Mechanisms
    • Correct/incorrect feedback displays appropriately
    • Feedback contains helpful chemistry information
    • Visual indicators work as expected
  5. Quiz Completion
    • Final score calculates correctly
    • Completion screen displays appropriately
    • Restart functionality works

Drag and Drop Functionality

  1. Drag Initiation
    • Items can be picked up with mouse click and hold
    • Touch events work on mobile devices
    • Visual indication of dragging state
  2. Drag Movement
    • Dragged items follow cursor/finger smoothly
    • Z-index ensures dragged item appears above other elements
  3. Drop Zones
    • Drop zones highlight when valid item hovers over them
    • Items can be dropped in designated zones only
    • Visual feedback when item is dropped
    • Items return to original position when dropped outside valid zones
  4. Answer Validation
    • Correct combinations are recognized
    • Incorrect combinations provide appropriate feedback
    • Multiple reagent combinations work correctly

Audio Functionality

  1. Sound Effects
    • Pick up sound plays when dragging begins
    • Drop sound plays when reagent is dropped
    • Correct answer sound plays on correct answer
    • Incorrect answer sound plays on incorrect answer
  2. Sound Controls
    • Mute button toggles all sounds
    • Sound preference persists between sessions (localStorage)
    • Initial sound state respects user's previous preference

Testing Tip

Use console.log statements to output values at key points in your code while testing. Remove these before submitting your pull request.

Cross-Browser Testing

The application should be tested across multiple browsers to ensure compatibility. At minimum, test in:

  • Desktop Browsers
    • Chrome (latest version)
    • Firefox (latest version)
    • Safari (latest version, if available)
    • Edge (latest version)
  • Mobile Browsers
    • Chrome for Android
    • Safari for iOS

Browser testing can be done using:

  • Real devices when available
  • Browser emulation tools (Chrome DevTools Device Mode)
  • BrowserStack or similar service for broader testing (when available)

Pay special attention to touch interactions when testing on mobile browsers or emulators, as they are critical to the drag-and-drop functionality.

Responsive Design Testing

Test the application at various screen sizes to ensure responsive behavior:

Key Breakpoints to Test

  • Mobile: 320px - 480px width
  • Tablet: 481px - 768px width
  • Small Desktop: 769px - 1024px width
  • Large Desktop: 1025px and above

Responsive Features to Check

  • Layout adapts appropriately to screen size
  • Text remains readable at all sizes
  • Images scale properly
  • Interactive elements (buttons, drag areas) are appropriately sized for touch on mobile
  • No horizontal scrolling occurs at any width

Use Chrome DevTools Device Mode or Firefox Responsive Design Mode to test different screen sizes:

  1. Open DevTools (F12 or Ctrl+Shift+I)
  2. Enter responsive design mode (Ctrl+Shift+M in Chrome)
  3. Test at various dimensions or select device presets

Accessibility Testing

Ensure the application is accessible to users with disabilities:

Manual Accessibility Checks

  1. Keyboard Navigation
    • All interactive elements can be reached using the Tab key
    • Focus indicators are visible on all interactive elements
    • Enter/Space keys activate buttons and controls
  2. Screen Reader Compatibility
    • All images have appropriate alt text
    • Form controls have associated labels
    • Semantic HTML is used appropriately
  3. Color and Contrast
    • Color is not the only means of conveying information
    • Text has sufficient contrast against backgrounds
    • UI is usable with high contrast mode

Automated Accessibility Testing

Use automated tools to identify potential accessibility issues:

  • axe DevTools (Chrome extension)
  • Lighthouse accessibility audit in Chrome DevTools
  • WAVE Web Accessibility Evaluation Tool

Performance Testing

Check that the application performs efficiently:

Performance Metrics to Monitor

  • Initial loading time
  • Time to interactive
  • Smooth animations (60fps target)
  • Responsiveness to user input

Performance Testing Tools

  • Chrome DevTools Performance panel
  • Lighthouse performance audit
  • Network throttling for simulating slow connections

Common Performance Issues to Watch For

  • Large image files causing slow loading
  • Inefficient JavaScript causing janky animations
  • Excessive DOM operations
  • Memory leaks during long quiz sessions

Reporting Test Results

When submitting a pull request, include information about your testing:

# Testing Notes

## Browsers Tested
- Chrome 115.0.5790.171 (Windows)
- Firefox 115.0.2 (Windows)
- Safari 16.5 (iOS)

## Devices/Resolutions Tested
- Desktop (1920x1080)
- iPhone 12 Pro (390x844)
- iPad (768x1024)

## Test Results
- All test cases passed on desktop browsers
- On mobile, drag and drop is working as expected
- Interactive elements appropriately sized for touch
- No console errors observed

## Known Issues
- None
